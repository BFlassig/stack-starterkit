# Security Policy

## Supported Versions

Security fixes are provided for the current `main` branch state.

| Version | Supported |
| --- | --- |
| main (current) | yes |
| older states | no |

## Reporting Vulnerabilities

Please do not report vulnerabilities publicly via Issues.
Do not post exploit details, stack traces, or sensitive environment data in public threads.

Preferred channels:

1. GitHub Security Advisory / private vulnerability report (if enabled)
2. Direct confidential contact with the repository maintainer

Maintainer contact:

- GitHub: https://github.com/BFlassig
- LinkedIn: https://www.linkedin.com/in/benjaminflassig/

## What to Include in a Report

- affected component
- reproduction steps
- expected vs actual behavior
- impact (for example data access, privilege escalation, DoS)
- optional fix or mitigation proposal
- environment details (OS, runtime, versions) if relevant

## Response Targets

- acknowledgement: within 3 business days
- initial technical assessment: within 7 business days
- coordinated disclosure after a fix is available

## Scope

This policy applies to code in this repository.
It does not cover issues in external services or third-party target environments.

## Disclosure Policy

- Vulnerabilities are handled confidentially until a fix is available.
- Public disclosure should be coordinated with the maintainer.
